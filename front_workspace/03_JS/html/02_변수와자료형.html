자바스크립트의 변수(Variable)

      변수 : 변하는 수
      상수 : 상시적으로 일정한 값을 가진 수

      * 메모리에 값을 계속 바꿔가며 저장할 수 있는 공간
      * 변수에 이름을 붙이고 원하는 값을 대입하면 필요한 곳에서 변수명을 호출하여 저장된 값을 사용할 수 있다.

      
변수이름(=변수명) 명명 규칙

      - 의미 있는 이름 작성(a, aaa, bb, jh, abc 등으로 짓지 말기)
      - 대, 소문자 구분
      - 특수문자는 _(언더스코어) 가능 (단 첫 글자 제외)
      - 숫자 사용 가능(단, 첫글자 제외)
      - 카멜 케이스(Camel Case) 표기법 사용
        [ 카멜 케이스(Camel Case) ]
        문장, 여러 단어를 변수명으로 작성할 때 이어지는 첫 번째 단어를 대문자로 표기
        변수명엔 띄어쓰기가 포함되선 안되기 때문
        ex) user name = userName

JavaScript 변수의 종류와 특징

      NEW 바구니A = '물건이나 숫자가 담겼습니다.';

      var = Variable
      var 바구니A = '물건이나 숫자가 담겼습니다.';

      var : 변수, 변수명 중복O (덮어쓰기 가능)
            덮어쓰기나 여러가지 문제가 생겨서 변수명 중복이 불가능
      
      let : 변수, 변수명 중복 불가능

      const : 상수, 변수명 중복 불가능, 값 변경 불가능

      1순위 : const(JS는 특정 요소를 선택해서 사용하는 경우가 많아 변수에 고정 시켜둠)
      2순위 : let
      3순위 : var

자바스크립트의 자료형

      자바스크립트는 변수 선언 시 별도의 자료형을 지정하지 않음
      -> 변수에 대입 되는 값(리터럴)에 의해서 자료형이 결정됨

      - 숫자는 '', "" 없이 표기
      - 문자열은 '', ""로 감싸서 작성
      - 참, 거짓을 나타내는 true, false는 '', "" 없이 표기
      - function(함수,기능)
      - undefined (정의되지 않은 변수) -> 변수가 선언 됐지만, 그 안에 값이 텅 비어 있는 상태
      - null (참조하는게 없음)
        -> document.getElementById("오타")
        -> id 값이 일치하는 요소가 없어 참조 할 수 없다.
